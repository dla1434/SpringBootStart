
1. ES Client 처리
	[에러]
		es에 message필드에 대해서 wildcard 문법으로 질의하는데 *push*는 되는데 *PushJob* 이런 경우는 안 됐다.

	[원인]

	[해결]
		> message 필드가 다음과 같이 text/keyword로 지정되어 있다면..
		> 질의 시 message.keyword 형태로 keyword를 붙여줘야 정상적으로 질의가 가능하다.
			>> 또는 message 필드에 대해서 mappper에서 keyword로 지정하면 keyword을 안 붙여서 keyword로 검색된다.
		"message": {
			"type": "text",
			"fields": {
				"keyword": {
				"type": "keyword",
				"ignore_above": 256
			}
			}
		},

		# 질의 퀴리
		{
		  "query": {
		    "bool": {
		      "must": [
		        {
		          "wildcard": {
		            "message.keyword": {
		              "wildcard": "*PushJob*"
		            }
		          }
		        }
		      ]
		    }
		  }
		}
